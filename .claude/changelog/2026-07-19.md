## Architecture Reminder
See `CLAUDE.md` at repo root for full picture. Key points:
- All Firebase/SMS/Firestore is **stubbed** in `fontend/src/lib/services.js` (localStorage-backed)
- The demo patient is always `uid: 'demo-patient'`, mapped to thread/session key `p1`
- Chat and session results both go through the same WebSocket server (`chat-server.js` port 3001) but use separate client singletons (`chatService` vs `sessionService`)
- `fontend/src/lib/mockData.js` — source of truth for exercises, patients, history (stands in for Firestore)
